## RESTful Services

RESTful服务也叫RESTful API，REST全称 **Representational State Transfer**代表性状态传输，本质上讲REST就是用来创建这些规范的HTTP服务的。我们使用简单的HTTP协议原则，去提供数据的增Create删Delete改Update查Read服务(CRUD也叫增删改查)。

现在来解释一个RESTful范例：假设有家叫Vidly的公司是做视频租凭业务的，有个用来管理用户信息的应用，在服务器上需要提供这么一个终端地址[http://vidly.com/api/customers]，客户端可以通过发送HTTP请求到这个终端来进行服务和对话。终端地址的组成：http或https开头，这取决于请求的类型，如果你希望数据在加密的形式下传输，就需要使用https; 域名vildly.com; 资源路径。请求的种类对应着要进行的操作，所有的HTTP请求都有所谓的动作或者方法，取决于它们的类型和请求目的。get用来获取数据，post发布数据，put更新数据，delete删除数据...现在以用户资源为例来解释每个方法：

1️⃣  ，我们向终端发送get请求，GET api

| 目的                                               | 方法   | 客户端Request                  | 解释                                                         | 服务器Response                                               |
| -------------------------------------------------- | ------ | ------------------------------ | ------------------------------------------------------------ | ------------------------------------------------------------ |
| 得到用户的的列表                                   | GET    | /api/customers                 | 注意这里的customers,代表我们操作的是用户集合，是请求所有用户列表 | [ {id:1,name:'whh'},{id:2,name:'lsd'},......]服务器返回的是用户的列表 |
| 如果我们需要一个单独的用户，就需要包含这个用户的id | GET    | /api/customers/1               | 注意这里的/1                                                 | {id:1,name:'whh'}      服务器返回单独的对象                  |
| 更新用户数据                                       | PUT    | /api/customers/1  {name:'zks'} | /1 这里需要包含需要更新的用户id,同时也需要在请求体中包括更新信息 | {id:2,name:'zks'}服务器根据请求体的值来更新数据              |
| 删除一个用户                                       | Delete | /api/customers/1               | 不用在请求体中包含用户的对象，因为删除只需提供用户id就可以了，这里的/1就提供过了 |                                                              |
| 创建一个用户                                       | POST   | /api/customers {name:'nml'}    | 因为是添加一个新用户，就不是操作特定的已存对象，资源地址就没有id(/1)，我们操作的是用户集合。我们向集合中添加一个新的用户，所以在请求体中包含用户对象 | {id:n,name:'nml'}        服务器得到对象并在集合中创建出来    |

这就是RESTful的范例，我们用一个有语义的地址来公开服务器资源，并且支持对该资源的增删改查等多种操作。
